# Как собрать APK через GitHub (без Android Studio)

Я подготовил всё необходимое. Теперь вам нужно просто «скормить» эти файлы Гитхабу, и он сам соберет приложение.

## Шаг 1: Создание репозитория
1. Зайдите на [GitHub.com](https://github.com/) и войдите в свой аккаунт.
2. Нажмите кнопку **New** (создать новый репозиторий).
3. Назовите его, например, `proxfier-mobile`.
4. Сделайте его **Public** или **Private** (как вам удобнее).
5. Нажмите **Create repository**.

## Шаг 2: Загрузка файлов
1. На странице репозитория нажмите **uploading an existing file**.
2. Перетащите **содержимое папки `mobile`** (не саму папку `mobile`, а всё, что внутри неё: `app`, `.github`, `build.gradle` и т.д.) в окно браузера.
3. Дождитесь окончания загрузки и нажмите **Commit changes**.

## Шаг 3: Автоматическая сборка
1. Сверху в меню репозитория нажмите на вкладку **Actions**.
2. Вы увидите запущенный процесс **Build Android APK**.
3. Нажмите на него и подождите 5–7 минут, пока он не станет зеленым (успех).

## Шаг 4: Скачивание готового файла
1. Когда сборка завершится, пролистайте страницу вниз до раздела **Artifacts**.
2. Там вы увидите файл **Proxfier-Mobile-Debug**.
3. Нажмите на него — скачается архив, внутри которого будет ваш готовый **app-debug.apk**.

**Всё! Этот файл можно сразу устанавливать на телефон.**
